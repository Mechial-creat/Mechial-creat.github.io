本体核心原理慢扫描电视(SSTV)

SSTV 是一种通过无线电波（通常是窄带音频信道）传输静态图像的方法。简单来说，它把图像的像素颜色和亮度信息转换成了特定频率的音频信号。

  1. 图像到音频的转换 (编码)就像传真机或拨号上网的猫（Modem）一样，SSTV 将图像逐行扫描：• 水平扫描：设备从左到右扫描图像的一行像素。• 频率调制 (FM)：每一个像素的颜色或亮度被映射到一个特定的音频频率。• 例如：在黑白模式下，1500Hz 代表黑色，2300Hz 代表白色。• 在彩色模式下（如 Robot36 模式），会分别编码红色、绿色和蓝色的色彩分量。• 同步脉冲：在每一行扫描结束时，会发出一个特定的低频信号（通常是 1200Hz），告诉接收端：“这一行结束了，请开始下一行。”
  2. 音频到图像的还原 (解码 - Robot36 的作用)当你用 Audacity（第一张图）打开这个音频时，你会看到密集的、有规律的波形（频谱图中呈现出一条条纵向的线条），这就是经过频率调制的音频数据。Robot36 软件的作用就是：1. 听：实时捕获音频频率。2. 识别模式：识别音频开头的 VIS（垂直指示信号），确定该音频使用的是哪种 SSTV 协议（Robot36 协议就是其中一种常见的彩色模式）。
  3. 还原像素：根据频率的高低，将声音实时还原回对应的颜色点，并按照同步信号一行一行地在屏幕上“画”出图片。

TE

通过共模攻击，利用给定的两个加密指数 e1 和 e2 以及密文 c1 和 c2，在相同的模数 n 下恢复出明文 m。具体步骤如下：

  1. 验证 gcd(e1, e2) = 1，满足共模攻击条件。

  2. 使用扩展欧几里得算法计算系数 s 和 t，使得 s*e1 + t*e2 = 1，得到 s = 1476540539，t = -381577012。

  3. 由于 t 为负数，计算 c2 模 n 的逆元，并计算 m = (c1^s * (c2^{-1})^{-t}) mod n。

  4. 将 m 转换为字节串，得到 flag。

这道题的核心是利用 MD5 算法的碰撞缺陷（Collision）。通过构造两段不同的二进制数据，使它们的哈希结果完全一致，从而绕过程序的逻辑校验。

以下是该挑战的原理总结与实操步骤：

一、 核心原理

  1. MD5 逻辑漏洞： MD5 是一种哈希函数，本应具有强抗碰撞性。但由于其内部算法逻辑（Merkle-Damgård 结构）存在缺陷，攻击者可以使用差分攻击，在已知前缀的基础上，快速寻找两个不同的块，使它们产生的哈希值相同。
  2. 代码校验绕过： 题目代码设置了两个关卡：
    • apple1 == apple2：检查输入是否完全相同。
    • hashlib.md5(apple1).hexdigest() == hashlib.md5(apple2).hexdigest()：检查哈希是否相同。 普通的 MD5 攻击（如彩虹表查库）无法解决这种“两端数据都要由你提供”的碰撞问题，必须使用专门的碰撞生成工具。

二、 解题步骤总结

第一步：准备工具

由于手动计算碰撞几乎不可能，需要使用如 Fastcoll 这样的专用工具。它能通过数学模型在亚秒级时间内计算出符合条件的二进制块。

三、 知识点延伸

bkcrack

下面把 bkcrack.exe（ZipCrypto）常用指令按“你在 CTF 里最常用的场景”整理一份（以 Windows/PowerShell 为例，命令行参数在 Linux/macOS 也一样）。

0）先看帮助与版本
.\bkcrack.exe –help
.\bkcrack.exe –version

1）列出 zip 里的文件、加密方式、压缩方式（最常用）
.\bkcrack.exe -L target.zip

用途：确认是不是 ZipCrypto、哪些 entry 加密、是否 Store/Deflate，文件名是否带路径。

2）已知明文攻击：用外部明文文件（核心）

当你知道某个加密文件的一段明文（≥8 bytes 连续）时：

.\bkcrack.exe -C target.zip -c secret.bin -p known.bin -o 0

参数含义：

-C：密文所在压缩包（Ciphertext archive）

-c：被加密的条目名（Ciphertext entry）

-p：已知明文文件（Plaintext file，外部文件）

-o：明文在目标文件里的偏移（offset）

常见做法：扫偏移 -o 0..N，或增加已知明文字节长度。

3）已知明文攻击：明文也在 zip 里（同一归档里取明文）

当明文文件就在压缩包里（通常是未加密 entry）：

.\bkcrack.exe -C target.zip -c secret.bin -P target.zip -p known.txt -o 0

-P：明文所在压缩包（Plaintext archive）

-p：明文 entry 名（Plaintext entry）

注意：这只适用于“明文 entry 的内容确实对应你要攻击的那段明文”。
不能拿 txt 明文去匹配 exe 密文（内容不同就找不到 keys）。

4）继续/恢复攻击（中断或想从某个进度继续）

当输出提示你可以 –continue-attack N：

.\bkcrack.exe … –continue-attack 108391

用途：跑了一半停了、或者你想接着搜剩余空间。

5）用 keys 解密导出文件（拿到 Keys 后必用）
.\bkcrack.exe -C target.zip -c secret.bin -k k0 k1 k2 -d out.bin

例子（你那题）：

.\bkcrack.exe -C 083222_CrackM3.zip -c flag.exe -k 60101051 4cba82cb 48eac20c -d flag_decrypted.exe

6）生成“伪加密/新 zip”（CTF 常见：想把 zip 变成可用密码/可直接解）

bkcrack 还有一些功能可生成新的归档或伪造内容（不同版本参数略有差异）。最稳的方式是先看 help：

.\bkcrack.exe –help | more

然后根据你版本里出现的选项（常见关键词：–make-zip / –create / –rebuild / –patch 等）来用。
（因为各版本命名有差异，我不硬写死，避免你复制后报“unknown option”。）

7）CTF 实战小抄（最常用三板斧）

看结构：

.\bkcrack.exe -L target.zip

跑 keys（已知明文）：

.\bkcrack.exe -C target.zip -c secret.bin -p known.bin -o 0

.\bkcrack.exe -C target.zip -c secret.bin -k k0 k1 k2 -d out.bin
